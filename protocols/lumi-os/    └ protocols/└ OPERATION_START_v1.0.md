OPERATION_START_v1.0（運用開始宣言・公式版）

目的
LUMI OS の GitHub 運用を正式開始し、A／B／C／D／E が完全に lumi-os の正本に基づいて稼働するための起点を定義する。

⸻

1. 運用開始の宣言

本日をもって：

「LUMI の OS・仕様・META・アルゴリズムの“唯一の正本”は
　GitHub lumina-bulige-dev / lumi-os に置く」

という憲法ルールを実装し、運用開始とする。

⸻

2. 適用対象（全クラス）

A：HQ

・唯一の自動整合性チェック権限
・採択した OS を lumi-os に書き込む唯一のレイヤー

B：INFRA

・lumi-os の OS / ALGO を唯一の正として参照
・独自ルール生成禁止
・貸付／預かり／投機／レバは永久禁止領域

C：PRODUCT

・UI／文言／誘導は lumi-os の OS 制約に従う
・OS が禁止する操作は UI でも許可しない

D：GTM

・コピー・外向けメッセージは lumi-os のMETA・法的境界に従う

E：DEEP

・源泉
・すべての内容は AI_E → A：HQ → lumi-os へ昇格
・正本化はしない（ドラフト層）

⸻

3. ディレクトリ構造（正式宣言）

/core-os
/meta
/rules
/algorithms
/protocols
/specifications
/changelogs

本構造を LUMI OS の 永続フォーマット とする。

4. 運用ルール

4.1 正本ルール（Canonical Rule）

「If it is not in lumi-os, it is not official.
　迷ったら lumi-os を見る。他はすべてドラフト。」

4.2 更新フロー

E → AI_E → A：HQ → lumi-os
（A で採択されたものだけ OS 化）

4.3 禁止

・B/C/D/E による OS 改変
・GitHub への直接書き込み（A 以外）
・自動補正
・専用領域越境

⸻

5. 本宣言の効力発生日

コミットされた瞬間から効力発生。
以後は lumi-os の記載が LUMI の唯一の OS として扱われる。
