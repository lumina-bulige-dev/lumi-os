HOME_SCREEN_PHILOSOPHY_OS v1.0

（LUMI ホーム画面の役割定義・正式 OS）

目的

ホーム画面は、ユーザーが 5 秒で「今日の自分の安全度」 を把握するための
LUMI OS のフロントラインである。

⸻

1. 必須 5 要素（A が定義する思想コア）

1）床（paket_bigzoon）

絶対に割りたくないライン。今日の許容度を即時可視化。

2）今日のポジション
	•	balance_total vs paket_bigzoon
	•	SAFE / WARNING / DANGER の 3 種判定

3）30日チャレンジの状態
	•	何日目か
	•	今日が SAFE_NULL day かどうか

4）心のブレーキ状態（ライト版）

NORMAL / TIRED / RED（詳細は別画面）

5）Aurora ルーペ入口（“見るだけ”）
	•	手数料の可視化のみ
	•	金は動かさない
	•	規制境界を完全遵守する UI

⸻

2. ホーム画面の役割

「今日の安全度を、5 秒でつかませる OS ダッシュボード」
フィンテック標準（残高画面中心）ではなく、
LUMI は 即死防止 OS のフロント として位置づける。

⸻

3. クラス別の影響

B：INFRA
	•	floor / risk / fatigue / SAFE_NULL 判定 API が必要
	•	/api/v1/os/reaction と連動する

C：PRODUCT
	•	ワイヤーフレームがここで固定
	•	禁止 UI（送金実行等）は出さない

D：GTM
	•	「今日の安全度 OS」がメッセージの主軸になる

⸻

4. A の最終一行

ホーム画面は金を動かさせる場所ではなく、
“今日の即死危険度を OS として見せる場所”である。
