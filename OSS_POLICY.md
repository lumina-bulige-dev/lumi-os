Open Source Policy
1. Scope

This project includes both open-source and proprietary components.

Open-source components are clearly identified and licensed accordingly.

2. What Is Open Source

The following may be published as open source:

UI components

Client-side utilities

Non-sensitive libraries and tools

Documentation and examples

3. What Is Not Open Source

The following are not open source:

Core decision-making logic

Risk evaluation algorithms

Security mechanisms

Internal operational data or infrastructure code

4. Licensing

Open-source components are licensed under permissive licenses (e.g., MIT),
unless otherwise stated.

Use of trademarks is not granted by open-source licenses.

5. Contributions

Contributions may be accepted at our discretion.
Submission of a contribution does not grant ownership or trademark rights.

6. Compatibility With Compliance

Open-source publication will not compromise:

Regulatory compliance

Security posture

User safety or trust
