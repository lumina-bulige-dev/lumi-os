## 0. Mission

F：GOV は「ユーザーの生活防衛」と「リーガル」を両立させる見張り台である。  
Heiankyo（行動ログ）と Aurora（送金比較）が、法律・規制・コンプラのラインを超えないよう常時監視し、リスクが見えたら A：HQ へ即フィードバックする。

## 1. Heiankyo（行動ログ）の法務・倫理チェック

### 1.1 対象（Scope）
- BULIGE Rank / 行動信用レイヤー
- Heiankyo ログ設計
- 行動指標・スコア算出ロジック
- ユーザー向けレポート・UI（行動評価の出し方）

### 1.2 チェックポイント
1) 個人情報・プライバシー
- 粒度が細かすぎないか（位置情報・健康情報などの過剰取得を避ける）
- 匿名化・ID化されているか（本名・電話番号と直結しない）
- 保存期間・削除ルールが決まっているか

2) 差別・不当な取り扱い
- 差別につながる属性（人種・宗教・病歴 等）を指標に入れていないか
- ユーザーの社会的地位を不当に下げるレポート文言になっていないか

3) 利用目的と説明責任
- 「何のためにログを使うか」がユーザーに説明されているか  
  例：アドバイス／レポートのため、現時点ではクレジット審査に用いない
- 同意画面・利用規約に目的が明記されているか

4) スコアの使い方
- 現段階で「金融機関の与信判断に直接連携しない」前提が守られているか
- 将来クレジット連携の可能性が出た場合は、金融・個人情報保護の専門家レビューを必須ステップに入れる

### 1.3 ワークフロー（F:CHECK_HEIANKYO）
- C：PRODUCT（および D：GTM が文言に関与する場合）が Heiankyo に関する変更案・新指標を作成したら、`F:CHECK_HEIANKYO` タグで F：GOV にレビュー依頼する。
- F：GOV の返答は以下 3 択：
  - ✅ OK：現状の規制・ポリシーに抵触なし
  - 🟡 要修正：文言／粒度／保存期間 等の部分修正で対応可能
  - 🔴 NG：根本見直しが必要／外部専門家レビュー推奨

## 2. Aurora（送金比較）の Fintech ライン監査

### 2.1 対象（Scope）
- Aurora：送金比較・シミュレーター機能全般
- Wise / 銀行 / 他サービス比較記事・LP
- 為替レート・手数料の計算ロジック

### 2.2 チェックポイント
1) 「比較ツール」から「投資助言」へ踏み出していないか
- Aurora のポジションは「送金コストの見える化・教育用ツール」
- 具体的な投資判断（買え／売れ／レバレッジ 等）を指示しない

2) 表示の透明性
- 前提条件（取得タイミング・レート種別）が明記されているか
- 「条件付き比較」になっているか（常に一択・断定を避ける）

3) 誤認リスク
- 「必ずお得」「ノーリスク」等の誇大・断定がないか
- 将来の為替や規制を見通したかのような記述をしない

4) 法規制ライン（原則）
- 「送金を代理する」「資金を預かる」構造になっていないか
- 「代理行為」「助言行為」にならないよう、情報提供とシミュレーションに留める

### 2.3 ワークフロー（F:CHECK_AURORA）
- C：PRODUCT / D：GTM が Aurora の新機能・画面・コピーを作成したら、`F:CHECK_AURORA` タグで F：GOV にレビュー依頼する。
- F：GOV の返答は以下 3 択：
  - ✅ 比較ツールとして許容範囲
  - 🟡 文言修正（断定表現・誤解を生むコピーの修正）
  - 🔴 構造上危険／外部専門家レビュー必須

## 3. A：HQ への早期警報フィードバック（常時）

### 3.1 役割
F：GOV は、A：HQ の意思決定より先に「ここ危ないかも」を通知する早期警報レーダーである。

### 3.2 フィードバック形式（3色）
- 🟢 GREEN：現時点で矛盾なし／通常運用で問題なし
- 🟡 YELLOW：グレー or 誤認リスクあり／文言・UI・注記で対応可能／再チェック推奨
- 🔴 RED：現行法・ガイドライン上そのままは高リスク／外部専門家レビュー必須／A 判断なく進めない

### 3.3 ログ（F:GOV_LOG）
F：GOV は重要な判定をログとして残す：
- 日付
- 対象（Heiankyo / Aurora / その他）
- 判定（GREEN / YELLOW / RED）
- 修正指示（あれば）

A：HQ はログを根拠に、進行を「停止／条件付きで進行／問題なし」で判断する。
